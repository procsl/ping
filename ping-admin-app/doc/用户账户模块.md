3. 仅邮箱校验
4. 全校验

## [添加账户接口](#添加账户接口)

POST /accounts

```JSON

{
	"account":"用户账号",
	"password":"用户密码",
	"name":"用户名(用于展示/区别于账号)// 未提供时和账户名相同/随机,根据配置",
	"typeComponent":"account|phone|email(用户注册账户的方式) // 必须",
	"token":"用户注册令牌(没有的用户可以不提供,主要用于初始化用户的权限/标识来源/提供注册权限等等)"
}
```

## [用户登陆](#用户登陆)

为了防止CSRF 禁止使用cookie

POST /tokens
```Json
{
	"account":"用户账户",
	"password":"用户密码",
	"typeComponent":"typeComponent// 登陆类型" 
}
```

// 如果登陆失败,则返回标准错误
// 如果登陆成功,需要返回认证的token
```
{
	"token":"token"
}
```
